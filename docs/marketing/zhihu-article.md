## 文章正文

最近半年我一直在折腾一件事：**做一个真正能帮我干活的 AI 助手**。

不是那种只能聊天的 ChatGPT wrapper，而是能自己去执行任务、能记住之前的对话、犯了错能自动改正的那种。

折腾了半年，踩了无数坑，最终做出来了。现在开源分享给大家。

### 背景

市面上的 AI 助手方案大致分两类：

1. **SaaS 产品**（Coze、Dify 等）：好用但数据不在自己手里，且重度定制困难
2. **开源框架**（OpenClaw、AutoGPT 等）：灵活但配置门槛高，需要很强的技术能力

我选择了第二条路 — 基于 OpenClaw 运行时，做深度增强。半年下来，从一个基础的聊天框架，做成了一个有 38 个技能、能自我进化、一键部署的完整平台。

### xyvaClaw 长什么样？

先说结论：**一行命令安装，浏览器里配置，装好就能用。**

```bash
git clone https://github.com/xyva-yuangui/XyvaClaw.git && cd XyvaClaw && bash xyvaclaw-setup.sh
```

安装过程完全自动化：检测依赖 → 弹出图形化向导 → 填入 API Key → 部署完成。

### 我觉得最有价值的几个设计

#### 1. 自我进化引擎

这是我花时间最多的部分。

普通 AI 助手是"无状态"的 — 每次对话都像第一次见面。xyvaClaw 有四层机制让它持续变聪明：

- **错误学习**：犯过的错自动记录到错误日志，下次遇到类似场景会主动规避
- **效果追踪**：每个决策的结果都会被评估，好的策略加强，差的策略淘汰
- **主动行动**：不只是被动等你问，它会主动发现可以帮你做的事
- **自我反思**：周期性地分析自己的行为模式，识别改进空间

实际感受：用一个月后，明显比刚装好时聪明。比如我经常让它查某类数据，第三周开始它会主动在早上帮我查好。

#### 2. 无损上下文

OpenClaw 原版的上下文管理是有损压缩 — 对话一长，早期信息就被丢弃了。

我自研了 lossless-claw 引擎：通过语义分段 + 增量摘要 + 关键实体追踪，在不超出上下文窗口的前提下保留所有关键信息。

效果：一个持续 2 小时的对话，中间讨论过的细节，到最后依然能准确引用。

#### 3. 五级模型容灾

用单个模型 API 最大的风险就是服务不稳定。DeepSeek 隔三差五就抽风，百炼也偶尔超时。

xyvaClaw 配置了五级自动切换：

```
DeepSeek V3.2 → Qwen3.5+ → Kimi K2.5 → DeepSeek Reasoner → Qwen3 Max
```

任何一级挂了，自动切到下一级，用户完全无感知。半年来我的 AI 助手没有一次因为模型故障停工。

#### 4. 飞书深度集成

如果你的团队用飞书，这个功能会很实用。

112 个 TypeScript 源文件，覆盖飞书几乎所有 API：消息、文档、多维表格、日历、审批、云盘、Wiki。

可以在飞书群里直接和 AI 对话，让它帮你写文档、查表格、发审批、安排日程。企业级的可靠性保障：消息去重、串行调度、超时看门狗、降级回退。

### 38 个内置技能

这些技能不需要额外安装，装好就能用：

- **浏览器自动化**：自动打开网页、填表单、截图
- **文档处理**：生成 Word、Excel、PPT
- **视频制作**：AI 辅助视频创作
- **量化选股**：A 股多因子筛选、技术分析
- **定时任务**：自动化日报、数据同步
- **小红书/社交媒体**：内容创作和自动发布
- ……还有很多

### 成本
