# 每日选股推送工作流

## 工作流概述

自动化每日 A 股选股流程，从交易日历确认到飞书推送的完整流水线。

## 工作流步骤

### 步骤 1: 获取交易日历
**目标**: 确认今天是交易日

**检查项**:
- 调用 Tushare `trade_cal` 接口
- 检查 `is_open` 字段
- 如果是非交易日，记录日志并退出

**输出**: `is_trading_day: true/false`

### 步骤 2: 运行量化策略选股
**目标**: 执行 quant-strategy-engine 选股

**参数**:
- 策略类型：技术面选股
- 筛选条件：
  - 非停牌
  - 成交量 > 100 万
  - 市值 > 50 亿
  - 符合选股因子

**输出**: `selected_stocks.json` (股票代码列表)

### 步骤 3: 生成选股报告
**目标**: 生成 Markdown 格式选股报告

**内容结构**:
```markdown
# 每日选股报告 - YYYY-MM-DD

## 执行摘要
- 选股数量：X 只
- 策略：技术面选股
- 市场状态：[描述]

## 选股列表
| 代码 | 名称 | 价格 | 涨跌幅 | 选股理由 |
|------|------|------|--------|----------|

## 详细分析
[每只股票的详细分析]

## 风险提示
[风险说明]
```

**输出**: `daily_report_YYYYMMDD.md`

### 步骤 4: 通过飞书发送报告
**目标**: 将报告发送到指定飞书群

**方式**:
- 使用 `smart-messenger` 技能
- 发送 Markdown 格式消息
- 或转换为图片发送

**参数**:
- `channel_id`: 目标群聊 ID
- `report_file`: 报告文件路径

### 步骤 5: 记录执行日志
**目标**: 记录工作流执行结果

**日志内容**:
- 执行时间
- 选股数量
- 发送状态
- 错误信息（如有）

**输出**: `workflow_log.json`

## 错误处理

### 非交易日
- 记录日志：`[SKIP] Today is not a trading day`
- 正常退出（exit 0）

### API 失败
- 重试 3 次（指数退避）
- 记录错误到 `error_log.json`
- 发送告警消息

### 选股结果为空
- 记录日志：`[INFO] No stocks selected today`
- 发送通知：「今日无符合选股条件的股票」
- 正常退出

## 执行时间

- **最佳执行时间**: 交易日 9:00-9:25（开盘前）
- **预计耗时**: 2-5 分钟
- **超时保护**: 10 分钟
