# Commerce Travel Shopper 功能介绍

## 1. 产品定位

Commerce Travel Shopper 是一个“代理执行 + 人工确认”的采购助手技能，用于：

- 机票比价（飞猪、去哪儿、携程、美团）
- 酒店比价（飞猪、去哪儿、携程、美团）
- 商品比价（淘宝、京东）

在无 API 场景下，技能以“页面截图 + OCR 解析 + 结构化比价”为核心方式，兼顾可落地性与风控安全。

本技能默认运行在 **飞书聊天会话** 中，因此所有关键流程都按“聊天消息 + 交互卡片 + 按钮确认”设计，确保非技术用户也能顺畅完成。

---

## 2. 核心能力

### 2.1 多平台比价
- 自动生成平台采集任务清单
- 对同一需求进行跨平台报价归一化
- 输出统一候选集与推荐结果

### 2.2 截图证据链
- 每条报价都绑定截图证据（evidence_id）
- 可追溯采集时间、来源平台、关键价格字段
- 支持人工复核，避免“黑盒推荐”

### 2.3 代理执行（受控）
- 可执行到下单页前步骤
- 支持填写订单草案（联系人、行程、规格）
- 不直接自动支付，关键动作需人工确认

### 2.4 风险控制
- 支持强制中止（ABORT）
- 价格异常、条款缺失、风控页面触发即止损
- 保留中止时快照和候选结果

### 2.5 飞书会话交互能力
- **卡片化引导**：每一步给出“下一步按钮”，用户无需记命令
- **双通道输入**：既支持按钮点击，也支持文本口令（如 `CONFIRM_PAY task_xxx`）
- **可视化进度**：在群聊/私聊中持续推送“当前阶段、剩余步骤、阻塞原因”
- **错误可恢复**：遇到验证码、超时、页面失败时，直接给出可执行补救按钮（重试/跳过/中止）

---

## 3. 关键流程（用户视角）

1. 用户在飞书发送任务（机票/酒店/商品）
2. 技能返回“任务确认卡片”（需求摘要 + 修改按钮 + 开始按钮）
3. 技能推送“平台采集清单卡片”（逐平台完成状态）
4. 代理执行截图采集并解析，飞书实时播报进度
5. 技能返回“比价结果卡片”（推荐项、次优项、风险说明、证据入口）
6. 技能返回“下单草案卡片”（提交前确认）
7. 技能返回“支付前确认卡片”（最终确认）
8. 完成或中止，并归档执行日志与证据索引

---

## 4. 人工确认节点

为避免误操作，以下节点必须人工确认：

- `CONFIRM_SUBMIT <task_id>`：允许提交订单
- `CONFIRM_PAY <task_id>`：允许发起支付
- `ABORT <task_id>`：立即停止并回滚

飞书按钮映射（建议）：
- `✅ 确认提交` -> `CONFIRM_SUBMIT`
- `💳 确认支付` -> `CONFIRM_PAY`
- `🛑 立即中止` -> `ABORT`

说明：按钮与文本口令等价，按钮优先用于降低误操作。

---

## 5. 输出结果

技能输出两类核心产物：

1. **比价报告**（Markdown/JSON）
   - 候选报价
   - 推荐排序
   - 风险提示（退改/取消/税费）
2. **执行日志**
   - 状态流转
   - 关键确认记录
   - 证据截图索引

3. **飞书消息记录**
   - 关键卡片快照（任务确认、比价结果、确认动作）
   - 用户操作轨迹（谁在何时点击了哪个按钮）

---

## 6. 适用边界

适合：
- 需要跨平台快速比价
- 希望 AI 代做重复操作但保留关键确认

不适合（首版不做）：
- 全自动无人支付
- 验证码绕过
- 高频爬取与大规模抓取

飞书交互边界：
- 不在会话中展示敏感支付凭据
- 关键确认必须由有权限用户点击/输入
- 超时任务自动降级为“等待人工续作”

---

## 7. 价值总结

Commerce Travel Shopper 不是单纯“问答助手”，而是可执行的交易流程代理：

- 能做：采集、分析、比价、下单前操作
- 你把关：提交和支付
- 结果可审计：每一步可回溯

在飞书场景下额外价值：
- 团队成员可以在同一会话中协同决策
- 通过卡片化流程降低学习成本与误触发率
